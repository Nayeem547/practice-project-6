Scope of var, let, and const:

var:

  • var has function-level scope. It's only limited to the function it is declared in, and it can also be hoisted to the top of the function.
  • When declared outside any function, it becomes a global variable, accessible throughout the entire program.
  • Variables declared with var are function-scoped, which means they are not restricted by block statements (e.g., if or for blocks).

Let:

  • let has block-level scope. It is confined to the block it's defined in (like loops or conditionals).
  • Variables declared with let are not hoisted to the top of the block and are only accessible within that block.
  • let allows you to create variables that are limited to a specific block, making it useful for preventing variable leakage.

const:

  • const also has block-level scope, similar to let.
  • The key difference is that const variables cannot be reassigned once they are given a value. However, the content of a const object (if it's an object or array) can be modified.

Use Cases of null and undefined:

null:

  • null represents the intentional absence of any object value or a specific object value that is intentionally empty.
  • It's often used to initialize variables or object properties when you want to indicate that they have no value yet.

undefined:

  • undefined represents the absence of a value in a variable that has been declared but not assigned any value.
  • It's also the default return value of functions without a return statement.
  • It can indicate a variable's initial state before a value is assigned.

REST API (Representational State Transfer):

REST API, or Representational State Transfer Application Programming Interface, is a set of architectural principles and constraints for designing networked applications. Here's a brief note:

  • REST is an architectural style for designing networked applications based on a client-server model.
  • It uses standard HTTP methods like GET (retrieve), POST (create), PUT (update), and DELETE (delete) to perform operations on resources.
  • Resources, represented as URLs, are the key abstractions in RESTful APIs.
  • REST APIs are stateless, meaning each request from a client to the server must contain all the information needed to understand and process the request.
  • Responses from the server are typically in JSON or XML format.
  • RESTful APIs are widely used for building web services, web applications, and mobile apps due to their simplicity and scalability.
